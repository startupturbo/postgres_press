PostgreSQL 8.0 Press Kit

PostgreSQL 8.0 Press Kit

米国ニューヨーク州ニューヨーク発：2005年1月19月 - PostgreSQL Global Development Groupはもっとも高機能なオープンソースデータベースと して認知されているPostgreSQL オブジェクト-リレーショナルデータベースの バージョン8.0をリリースしました．今回のリリースでは，もっとも高価な商 用データベースでしか使えなかった機能が実装されており，ユーザとソフトウェ アベンダーの双方でPostgreSQLの利用が確実に進んでいくものと期待されます．

PostgreSQL 8.0はスケーラビリティ，機能，性能において大きな進歩を遂げた だけでなく，オープンソースソフトウェアとしての比類のない開発速度を実証 したといえます．Red Hat，富士通, Afilias, (株)SRA, 2nd Quadrant, Command Prompt Inc.などの10数社および数百人の個人開発者が，8.0に対して かつてないほどの大量の機能追加を行いました．

富士通 ソフトウェア事業本部 ミドルウェアプラットフォーム事業部OSS データベース プロジェクト部長の仲沢孝之氏はこう述べています．「多くの PostgreSQLの新しいユーザにとって，こういった新しい機能が魅力的なものに 写るに違いないと私たちは，考えています．富士通は，PostgreSQLの開発に費 用面での貢献ができたこと，またPostgreSQLコミュニティと一緒に作業できた ことを誇りに思っています．これからも私たちはPostgreSQLを最先端のデータ ベース管理システムとして成長させる作業に協力していくことをお約束します」

PostgreSQLの新機能:

  • Windowsのネィティブサポート: PostgreSQLはエミュレーションな しにそのままWindowsで動くようになり，性能面で以前に比べると大幅な改善 がされています．このことにより，PostgreSQLはソフトウェア会社，企業ユー ザ，Windows開発者にとって商用データベースにとって代る非常に魅力的な代 替候補となりました．
  • セーブポイント：SQL標準であるセーブポイントが実装され，トラ ンザクション全体ではなくその一部のみをロールバックできるようになりまし た．複雑で，エラー回復が伴うトランザクションを必要とするビジネスアプリ ケーションの開発者に恩恵をもたらす機能です．
  • ポイント・イン・タイム・リカバリ：ポイント・イン・タイム・ リカバリにより，あらかじめ自動的かつ継続的に保存されたアーカイブ・トラ ンザクション・ログからデータを完全に復旧できるようになりました．毎時あ るいは日々のバックアップに頼ることなく，データの保全性に神経を使うよう な用途にもPostgreSQLがようやく使えるようになりました．
  • テーブルスペース：テーブルスペースは，ギガバイト級のデータ ウェアハウスシステムの管理者にとってなくてはならない機能です．テーブル スペースを使えば大きなテーブルやインデックスを別々のディスク装置やRAID 装置に作成することができるので，問い合わせの性能が向上します．
  • メモリとI/Oサブシステムの改善：Adaptive Replacement Cache (ARC)アルゴリズム，バックグラウンド・ライター・プロセス，VACUUM遅延機 能により，ディスクとメモリの利用方法が最適化されました．その結果、デー タベースへの負荷が突発的に高くなった時でも，負荷が予測しやすくなるとと もに安定した性能が得られるようになりました．

Hyperic有限会社のチーフアーキテクトJavier Soltero氏はこう述べていま す．「PostgreSQL 8.0によって，我々のHQモニター製品に必要な高度の同時実 行性能とスループットが得られました．PostgreSQL 8.0がWindowsをネィティ ブにサポートしたということは，我々にとってはPostgreSQLを製品に同梱でき るようになったことを意味します．その結果，スケーラビリティと性能で実績 を持ち，かつ商用利用においても自由に使えるライセンスを持つPostgreSQLの メリットを我々の製品は享受できるようになりました．」

PostgreSQLに追加された多くの機能に加え，PostgreSQLの利用環境は，こ こ1 年ほどの間にアドオンソフトやオプションソフトの開発が加速したことに よって改善されてきました．レプリケーションツールのSlony-Iとコネクショ ンプーリング/仲介ユーティリティpgpoolは，可用性を高めたサーバ群を構築 するためにすでに使われています．PL/Java, PL/PHP, PL/Perlなどのストアド・ プロシジャ言語が新規に開発あるいは拡張されています．一方，多数の WindowsユーザをサポートするためにNpgsqlやPGsqlClientといった.NETデータ プロバイダが改良されました．

PostgreSQL 8.0の新機能の詳細なリストはプレスリリースのページにあります．

PostgreSQLについて：PostgreSQLはカリフォルニア大学バークレー 校で開発が始まり，その後約20年ほどの間に積み重ねられた数百人の開発者の 成果を集大成したものです．トランザクション，関数，トリガ，副問い合わせ などのエンタープライズレベルの機能を長年にわたってサポートしてきた結果， PostgreSQLはビジネス規模の大きい多くの企業や政府機関で採用されています． PostgreSQL はBSDライセンスの下で配布されています．商用，非商用を問わず， 無償での利用と配布が許可されています．

このバージョンで追加された機能

上で説明したような大きな新機能の他に，PostgreSQLの既存ユーザが強い関心 を持つと思われる多くの新機能や改良点があります．

  • 関数：PostgreSQLの関数で，$記号を用いるヒアドキュメント形式がサポート されました．これにより，煩雑なネストした単一引用符を使う必要がなくなり ました．更に，新しいセーブポイント機能により，PL/pgSQL関数でEXCEPTION 句を使用して限定的な例外を扱えるようになりました．
  • データベース設計：データベース管理者はALTER TABLEを使って列のデータ型 をその場で変更することができるようになりました．SQL標準にもとづいて複 合データ型の列を定義できるようになりました．すべてのデータベースオブジェク トにおいて，CHANGE OWNERコマンドがサポートされました．
  • データのインポートとエキスポート：COPYコマンドにおいて，一般的なComma Separated Value(CSV)形式がサポートされました．その結果，ほかのソフトか らデータのインポート，エキスポートが容易になりました．
  • オプティマイザの改良：クエリ・プランナとエクゼキュータにおいて，互換性 のあるデータ型の間で（値を比較する際に）インデックスが使用できるように なりました．これによって定数の明示的なキャストを行う必要性が減少してい ます．また，インデックスの効率的な利用，OR節のより良い最適化，ANALYZE で行われるサンプリングの改良，より高速なTRUNCATE，PREPARED問い合わせ用 のクエリプランの改良が行われています．このほか，たくさんの細かな改良が 行われていますが，スペースの関係で省略します．
  • ログの取得：postgresql.confの設定により，PostgreSQLのログの自動ローテー ションができるようになりました．また，管理者によるログの行接頭辞設定機 能，特定の種類のSQL文をログする機能，接続の切断のログ機能など， PostgreSQLのログはより自由に設定できるようになりました．
  • バックアップ：可搬性のあるPostgreSQLのバックアップツールであるpg_dump は大幅に書き直されました．開始，終了時刻の記録機能，またバックアップファ イルにおけるパーミッションや依存関係の問題を解決しました．

もちろん他にもたくさんの変更や改良があります．詳細については，リリースノートRelease Notesを参照してください．
